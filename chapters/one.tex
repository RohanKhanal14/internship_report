\subsection{Introduction}

The rapid growth of web technologies has significantly transformed how modern software applications are designed and developed. Today’s applications are expected to be interactive, scalable, secure, and responsive across different devices. This shift has increased the demand for full stack developers who can work on both the frontend and backend of web systems and understand how all components integrate into a complete application.

DevOps is not merely a toolset but a philosophy that emphasizes automation, monitoring, version control, infrastructure as code (IaC), containerization, and cloud-based deployment. Combined with cloud computing platforms such as Amazon Web Services (AWS), DevOps enables organizations to build scalable, resilient, and highly available systems with faster release cycles and improved reliability. \cite{CI/CD}

During my internship at Genese Solution Pvt. Ltd., Lalitpur, Nepal, I worked as a Cloud and DevOps Engineering Intern, where I gained hands-on experience in implementing DevOps practices within real-world production environments. The organization specializes in cloud-based infrastructure solutions, automation, and scalable system design. Throughout the internship period, I was actively involved in designing and implementing CI/CD pipelines, managing AWS resources, automating deployment processes using AWS SAM (Serverless Application Model), configuring IAM roles and policies, and working with services such as AWS Lambda, CodeBuild, CodePipeline, S3, CloudFormation, EventBridge, and CloudWatch.

One of the most valuable aspects of this internship was understanding how different layers of an application communicate with each other. I learned how to structure backend projects properly, organize routes and controllers, manage middleware, design normalized database schemas, and ensure secure data transmission between client and server. Working in a collaborative environment also strengthened my skills in Git-based version control, debugging, and writing clean, maintainable code.

This internship helped me connect theoretical concepts learned in academic courses with real-world software development practices. It improved my understanding of software development lifecycle (SDLC), system design principles, REST architecture, and performance optimization in full stack applications.

\subsection{Problem Statement}

While developing modern web applications, several practical challenges can arise. Many traditional systems suffer from tightly coupled architectures, poor database design, and weak security mechanisms. Without a structured development approach, applications may become difficult to maintain, scale, or extend.

During full stack development, common technical challenges include:

\begin{itemize}
  \item Designing a clear separation between frontend and backend logic
  \item Managing database relationships and optimizing queries
  \item Implementing secure authentication and authorization mechanisms
  \item Handling asynchronous operations in backend services
  \item Managing application state effectively in frontend applications
  \item Ensuring proper validation and error handling
\end{itemize}

Organizations aiming to build cloud-based applications often struggle with:
\begin{itemize}
  \item Automating deployments efficiently
  \item Managing infrastructure securely
  \item Maintaining environment consistency
  \item Implementing continuous integration and continuous deployment (CI/CD)
  \item Managing multiple microservices or serverless functions efficiently
  \item Ensuring security and least-privilege access using IAM policies
\end{itemize}

Specifically, in serverless monorepo architectures, challenges arise in:
\begin{itemize}
  \item Handling shared dependencies (common layers)
  \item Triggering selective deployments
  \item Avoiding unnecessary rebuilds
  \item Maintaining version control for multiple Lambda functions
  \item Ensuring proper role-based access control
  \item Managing S3 artifacts and CloudFormation packaging
\end{itemize}

Therefore, there was a need to design and implement a structured DevOps workflow that would:
\begin{itemize}
  \item Automate build, package, and deployment processes
  \item Enable scalable serverless architecture
  \item Reduce manual intervention
  \item Improve deployment reliability
  \item Enforce security best practices
  \item Optimize cloud resource usage
\end{itemize}

\subsection{Scope and Limitation}
\textbf{Scope}
The scope of this internship primarily covered:
\begin{itemize}
  \item Frontend development using React.js with reusable component design
  \item Backend API development using Node.js and Express.js
  \item PostgreSQL database design and query implementation
  \item Implementation of CRUD operations
  \item Authentication and authorization mechanisms
  \item Git-based version control and collaborative development
  \item Basic deployment and application testing
\end{itemize}

The experience provided exposure to real development environments where applications are built based on actual business requirements. It also improved my understanding of how to write production-level code and maintain coding standards within a team.

The internship is directly related to academic subjects such as Software Engineering, Database Management Systems, Web Technology, and Object-Oriented Programming.

\textbf{Limitation}
Despite the extensive exposure, certain limitations were present:
\begin{itemize}
  \item The internship duration was limited.
  \item Access to highly sensitive production data was restricted.
  \item Exposure to large-scale enterprise architectures was limited.
  \item The primary focus remained on the PERN stack rather than multiple technology stacks.
\end{itemize}

Some advanced topics such as large-scale distributed systems, enterprise-level DevOps automation, and high-availability system design were beyond the scope of this internship.
