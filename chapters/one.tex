\subsection{Introduction}
Throughout the internship, I was involved in developing and maintaining full stack web applications based on real client requirements. On the frontend, I built reusable React components and designed responsive user interfaces to improve user experience. On the backend, I developed RESTful APIs using Node.js and Express.js \parencite{Fielding2000}, implemented authentication mechanisms using JWT \parencite{Jones2015}, and handled database interactions through structured queries in PostgreSQL.

The rapid growth of web technologies has significantly transformed how modern software applications are designed and developed. Today’s applications are expected to be interactive, scalable, secure, and responsive across different devices. This shift has increased the demand for full stack developers who can work on both the frontend and backend of web systems and understand how all components integrate into a complete application.

Full stack development allows a developer to handle the entire workflow of a web application, from designing user interfaces to building server-side logic and managing databases. During my internship at \textbf{Information Care Pvt. Ltd.}, I worked as a \textbf{Software Development Intern (Full Stack Web Developer)} where I primarily used the PERN stack (PostgreSQL, Express.js, React.js, and Node.js). This stack enabled me to work with JavaScript across the application while also gaining experience in relational database management using PostgreSQL.

One of the most valuable aspects of this internship was understanding how different layers of an application communicate with each other. I learned how to structure backend projects properly, organize routes and controllers, manage middleware, design normalized database schemas \parencite{Codd1970}, and ensure secure data transmission between client and server \parencite{McGraw2006}. Working in a collaborative environment also strengthened my skills in Git-based version control, debugging, and writing clean, maintainable code.

This internship helped me connect theoretical concepts learned in academic courses with real-world software development practices. It improved my understanding of software development lifecycle (SDLC) \parencite{Sommerville2010}, system design principles \parencite{Newman2015}, REST architecture \parencite{Fielding2000}, and performance optimization in full stack applications \parencite{Beck2001}.

\subsection{Problem Statement}

While developing modern web applications, several practical challenges can arise. Many traditional systems suffer from tightly coupled architectures, poor database design, and weak security mechanisms \parencite{McGraw2006}. Without a structured development approach, applications may become difficult to maintain, scale, or extend \parencite{Sommerville2010}.

During full stack development, common technical challenges include:

\begin{itemize}
  \item Designing a clear separation between frontend and backend logic \parencite{Newman2015}
  \item Managing database relationships and optimizing queries \parencite{Codd1970}
  \item Implementing secure authentication and authorization mechanisms \parencite{McGraw2006}
  \item Handling asynchronous operations in backend services
  \item Managing application state effectively in frontend applications
  \item Ensuring proper validation and error handling
\end{itemize}

In PERN stack applications specifically, developers must ensure that React components efficiently consume APIs, Express routes are well-structured, PostgreSQL schemas are normalized, and Node.js handles concurrent requests properly. Without proper design practices, issues such as slow performance, redundant code, and security vulnerabilities can occur \parencite{Humble2010}.

Therefore, it was necessary to follow a structured and practical approach to build scalable and secure full stack applications that maintain code quality and performance standards.

\subsection{Objectives}

The main objective of this internship was to gain practical experience in full stack web development using the PERN stack in a professional environment. The specific objectives were:

\begin{itemize}
  \item To design and develop dynamic web applications using PostgreSQL, Express.js, React.js, and Node.js.
  \item To implement RESTful APIs following proper architectural practices.
  \item To design efficient and normalized relational database schemas.
  \item To implement authentication and authorization using JWT.
  \item To improve problem-solving, debugging, and code optimization skills.
  \item To understand teamwork, version control, and real-world development workflows.
\end{itemize}

\subsection{Scope and Limitation}

\textbf{Scope}

The internship mainly focused on practical full stack development tasks, including:

\begin{itemize}
  \item Frontend development using React.js with reusable component design
  \item Backend API development using Node.js and Express.js
  \item PostgreSQL database design and query implementation
  \item Implementation of CRUD operations
  \item Authentication and authorization mechanisms
  \item Git-based version control and collaborative development
  \item Basic deployment and application testing
\end{itemize}

The experience provided exposure to real development environments where applications are built based on actual business requirements. It also improved my understanding of how to write production-level code and maintain coding standards within a team.

The internship is directly related to academic subjects such as Software Engineering, Database Management Systems, Web Technology, and Object-Oriented Programming.

\textbf{Limitation}

Despite the valuable learning experience, there were certain limitations:

\begin{itemize}
  \item The internship duration was limited.
  \item Access to highly sensitive production data was restricted.
  \item Exposure to large-scale enterprise architectures was limited.
  \item The primary focus remained on the PERN stack rather than multiple technology stacks.
\end{itemize}

Some advanced topics such as large-scale distributed systems, enterprise-level DevOps automation, and high-availability system design were beyond the scope of this internship.
